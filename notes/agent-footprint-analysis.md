## Framing: agents are not skills — there is no catalog budget, so this is anti-drift, not budget-relief

Before any recommendation, state the asymmetry that governs everything below. **Skills** demote out of the active listing via the umbrella shape because there is a documented `skillListingBudgetFraction` (~1% ≈ 2K tokens), an overflow behavior, and a `/doctor` readout — that is what made Waves 1–3 a measurable budget win. **Agents have no documented analog.** The agent catalog has no documented count cap, no per-byte budget, and no documented overflow behavior. The total agent-`description` payload today is **≈26,237 chars (~6.5K tokens) across 129 agents, avg ~203/agent**; *if* the skill ~1% rule applied it would be ~3× over, but no such rule is documented for agents, so there is nothing to cite as a hard limit.

The honest consequence — and the direct answer to "how can agents be demoted, if it can be done":

- **Agents cannot be "demoted" out of a catalog the way skills can.** There is no umbrella-shape equivalent that hides an agent from the catalog while keeping it invocable. The lever set is different and smaller:
  1. **Consolidation / parametrization** — collapse N byte-near clones into one axis-switched agent (the `target-locations-finder` Option-D precedent). Reduces the registered agent *count*.
  2. **Inlining single-call thin agents** — fold a deterministic, single-caller agent body back into its sole caller (a skill or orchestrator agent). Removes one catalog entry *and* one spawn round-trip.
  3. **Description trimming / model-tier hygiene** — `/trim-descriptions` shrinks the only metric that even arguably matters (the `description` payload); tier reconciliation is hygiene, not reduction.
- **Why do it at all, then:** the win is **code-health / anti-drift**, not proven budget necessity. N byte-identical clones *drift* — persistence-spec's own copy of the `target-locations-finder` skeleton had already drifted before consolidation. Removing redundant clones removes the surface where drift hides. This is real value, but it is **measurement-gated**: do the free/low-risk moves first, then measure (e.g. via `/doctor`-style accounting) before committing to anything that carries rewiring blast-radius, exactly mirroring the skills-footprint "most-reversible-and-diagnostic-first" discipline.

A second invariant, repeated across nearly every verdict and worth stating once at the top: **a merge only makes sense when the skeleton is byte-near and the divergence is *data* (a per-axis table/config), not *control* (a per-axis `if layer==X` body).** The `target-locations-finder` collapse worked because the skeleton was byte-identical and only a per-layer path *table* differed. Every REJECT below fails this test — the control flow itself forks per axis, producing a "god-agent" with mutually-exclusive branches and no shared middle. A second, independent hard blocker recurs: **frontmatter `skills:` lists are static and cannot be parametrized per-invocation**, so a spec-core-homed merged agent must auto-load the *union* of every layer's `patterns` umbrella on every call — a permanent active-skills-footprint *regression* that cuts directly against the very project that motivated this work.

## (D) Inlining candidates

Inlining folds a deterministic, **single-caller** agent body back into its sole caller, removing one catalog entry *and* one spawn round-trip. The discriminator is mechanical-Bash-only-and-no-new-tool-grant vs. judgment-bearing-multi-file-Read/Write/Edit.

**Do now (low risk, saved = 1 each):**
- **`domain-spec:test-package-preparer` → `init-domain` skill — saved 1, low.** Sole caller. ~8-command fixed Bash sequence (existence-check-then-mkdir/touch for `tests/`, `tests/conftest.py`, `tests/unit/__init__.py`) + 2 cheap arg guards. No Read/Write/Edit, no pattern docs. init-domain already declares `allowed-tools: Bash, Agent` → zero new grants, no fan-out lost. Only loss is a cosmetic per-line confirmation the skill already suppresses.
- **`domain-spec:domain-bootstrapper` → `init-domain` skill — saved 1, low.** Sole caller. Deterministic Bash: ensure `<domain_dir>` + `__init__.py`, `find` the plugin `modules/shared`, `cp -r` if absent. The optional self-derivation of `<domain_dir>` from `src/` is **dead code** (init-domain always passes it explicitly) → drop that branch on inlining. Folds into the existing Bash grant; slightly more involved than test-package-preparer but still a fixed sequence.
- **`persistence-spec:integration-test-package-preparer` → `init-persistence` skill — saved 1, low.** Sole caller. Same shape as test-package-preparer (4 existence-check-then-mkdir/touch for `tests/`, `tests/conftest.py`, `tests/integration/`, `tests/integration/conftest.py`), pure Bash. init-persistence already declares `Bash, Agent` and already does inline Bash mkdir/touch in Steps 4–5 — folding Step 7 in matches the skill's existing style, adds no tools.

**Do with care (saved = 1, moderate):**
- **`domain-spec:package-preparer` → `code-generator` agent — saved 1, moderate.** Sole caller. Core is a 3-line cumulative mkdir+touch loop over `<package_path>` segments + 3 cheap preconditions; code-generator already has Bash and already does inline Bash (`ls`-globs in Step 5). **Moderate** because code-generator is a large multi-worker orchestrator and the agent boundary currently isolates the "refuse if `/init-domain` hasn't run" contract (distinct abort messages pointing back to `/init-domain`) — the orchestrator must reproduce those guard messages inline. Net-positive but a slightly larger surface than the pure-init cases.

**Keep the agent (saved = 0, high risk — do NOT inline):** `persistence-spec:database-session-scaffolder` (byte-faithful Read+Write copy + 3-case `extras/__init__.py` re-export patch, ~80 lines; skill has only `Bash, Agent`), `persistence-spec:unit-of-work-scaffolder` (~135 lines of containers.py patch judgment + fail-fast ambiguity rules), `persistence-spec:query-context-scaffolder` (largest; adds conftest fixture insertion), `rest-api-spec:serializers-copier` (~100 lines, byte-copy + disk-scan star-aggregator render; init-rest-api lacks `Read`), `rest-api-spec:test-fixtures-preparer` (pattern-doc-driven, Read/Edit/Write-heavy), `messaging-spec:consumer-scaffolder` (~270 lines, spec-parsing + collision rules + aggregator patch + constants append; frontmatter-loads two umbrellas).
